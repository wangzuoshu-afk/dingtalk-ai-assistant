# 钉钉AI助手使用指南

欢迎使用钉钉AI助手！本指南将帮助您充分利用AI助手的各项功能，提升工作效率。

## 功能概览

钉钉AI助手是一个基于OpenAI GPT-4和Whisper模型的智能助手，它可以：

- **回答AI相关问题**：无论是技术原理、应用场景还是行业动态，AI助手都能为您提供专业的解答。
- **生成详细报告**：当您需要深入了解某个AI主题时，只需一句话，AI助手就能为您生成一份结构完整的PDF报告。
- **每日资讯推送**：每天早上9点，AI助手会自动向群聊推送最新的AI行业新闻和技术动态，让您第一时间掌握行业脉搏。
- **语音交互**：支持语音消息输入，AI助手会自动将语音转换为文字并进行回复，让交流更加自然便捷。

## 基础使用

### 1. 文字对话

在钉钉群聊中，您可以通过@机器人来与它对话。例如：

```
@AI小助手 什么是Transformer模型？
```

AI助手会立即回复您，提供关于Transformer模型的详细解释。您也可以与机器人进行单聊，无需@即可直接发送消息。

**对话示例**：

- **用户**：@AI小助手 GPT-4和GPT-3.5有什么区别？
- **AI助手**：GPT-4相比GPT-3.5在多个方面都有显著提升。首先，GPT-4的参数规模更大，这使得它在理解复杂问题和生成高质量回复方面表现更好。其次，GPT-4支持多模态输入，可以处理图像和文字的组合...

### 2. 生成报告

当您需要对某个AI主题进行深入了解时，可以请求AI助手生成一份详细报告。只需在消息中包含"报告"、"详细"、"分析"等关键词，AI助手就会自动识别并生成报告。

**使用示例**：

```
@AI小助手 生成一份关于大语言模型在医疗领域应用的详细报告
```

AI助手会开始生成报告，这个过程可能需要几十秒到一分钟的时间。生成完成后，AI助手会回复您，并提供报告的摘要。在生产环境中，报告会上传到云存储，您可以通过链接下载完整的PDF文件。

**报告内容结构**：

- **标题**：报告的主题
- **摘要**：报告的核心内容概述
- **引言**：背景介绍和问题提出
- **主体内容**：分多个章节详细阐述
- **结论**：总结和展望

### 3. 语音交互

AI助手支持语音消息输入，让您可以更自然地与它交流。需要注意的是，由于钉钉的限制，语音消息只能在与机器人的单聊中使用，在群聊中@机器人发送语音消息是无法被识别的。

**使用步骤**：

1. 打开与AI助手的单聊窗口。
2. 按住语音按钮，说出您的问题，例如"介绍一下强化学习的基本原理"。
3. 发送语音消息。
4. AI助手会自动将您的语音转换为文字，并进行回复。

**语音识别示例**：

- **您说**：（语音）"什么是神经网络？"

## 高级功能

### 1. 多轮对话

AI助手支持多轮对话，它会记住您之前的问题和回复，从而提供更加连贯和个性化的服务。对话历史会保留最近10轮对话，超过10轮后，最早的对话会被自动清除。

**多轮对话示例**：

- **用户**：@AI小助手 什么是深度学习？
- **AI助手**：深度学习是机器学习的一个分支，它使用多层神经网络来学习数据的层次化特征表示...
- **用户**：它和传统机器学习有什么区别？
- **AI助手**：深度学习与传统机器学习的主要区别在于特征提取方式。传统机器学习需要人工设计特征，而深度学习可以自动从原始数据中学习特征...

### 2. 触发关键词

AI助手会根据您消息中的关键词来判断是否需要生成详细报告。以下是一些触发报告生成的关键词：

- 报告
- 详细
- 分析
- 深入
- 研究

如果您只是想进行简单的问答，请避免使用这些关键词。

### 3. 自定义推送时间

如果您希望调整每日资讯的推送时间，可以联系管理员修改环境变量`DAILY_NEWS_TIME`。例如，将推送时间改为早上8点，只需将该变量设置为`08:00`。

## 使用技巧

### 1. 提问技巧

为了获得更好的回复，建议您在提问时：

- **明确具体**：避免过于宽泛的问题，例如"告诉我关于AI的一切"。更好的提问方式是"GPT-4在自然语言处理中有哪些应用？"
- **提供上下文**：如果问题涉及特定场景，请提供相关背景信息。例如"我正在开发一个聊天机器人，应该选择哪种AI模型？"
- **分步提问**：对于复杂问题，可以分成多个小问题逐步提问，利用多轮对话功能获得更深入的解答。

### 2. 报告生成技巧

生成报告时，建议您：

- **主题明确**：清楚地说明报告的主题，例如"生成一份关于计算机视觉在自动驾驶中应用的报告"。
- **指定范围**：如果希望报告聚焦于某个特定方面，可以在请求中说明，例如"重点分析技术挑战和解决方案"。
- **耐心等待**：报告生成需要一定时间，请耐心等待。如果超过2分钟仍未收到回复，可以重新发送请求。

### 3. 语音输入技巧

使用语音输入时，建议您：

- **环境安静**：在安静的环境中录制语音，避免背景噪音干扰识别准确率。
- **语速适中**：说话速度不要过快或过慢，保持自然的语速。
- **发音清晰**：尽量使用标准普通话，避免方言或口音过重。
- **单聊使用**：记住，语音消息只能在与机器人的单聊中使用，在群聊中无法识别。

## 注意事项

### 1. 隐私保护

AI助手不会主动记录或存储您的个人信息。对话历史仅保存在内存中，用于维持多轮对话的连贯性，服务重启后会自动清除。但请注意，您发送的消息会通过OpenAI API进行处理，请遵守OpenAI的使用政策，不要发送敏感或违规内容。

### 2. 使用限制

AI助手基于OpenAI API构建，因此受到API的速率限制和配额限制。如果短时间内发送大量请求，可能会遇到限流或配额不足的情况。建议合理使用，避免频繁发送无意义的消息。

### 3. 内容准确性

AI助手的回复基于大语言模型生成，虽然在大多数情况下是准确的，但仍可能存在错误或不准确的信息。对于关键决策或专业问题，建议您参考权威资料或咨询专业人士。

### 4. 语音识别准确性

语音识别的准确性受多种因素影响，包括录音质量、发音清晰度、背景噪音等。如果识别结果不准确，建议您使用文字输入，或者重新录制语音消息。

## 常见问题

**Q1：AI助手没有回复我的消息怎么办？**

A1：请检查以下几点：
- 在群聊中，是否正确@了机器人？
- 服务是否正常运行？可以访问健康检查接口确认。
- 是否遇到了API限流或配额不足？可以稍后再试。

**Q2：如何清除对话历史？**

A2：对话历史会在服务重启后自动清除。如果希望立即清除，可以联系管理员重启服务。

**Q3：可以让AI助手学习我们公司的知识吗？**

A3：当前版本的AI助手不支持自定义知识库。如果您有这方面的需求，可以考虑集成向量数据库（如Pinecone或Weaviate），实现基于企业知识库的问答功能。

**Q4：资讯推送的内容可以自定义吗？**

A4：当前版本的资讯推送内容是自动获取的AI相关新闻。如果您希望自定义推送内容，可以修改`utils/news_utils.py`文件中的新闻获取逻辑。

**Q5：生成的PDF报告在哪里？**

A5：在当前演示版本中，PDF报告保存在服务器本地。在生产环境中，建议将PDF上传到云存储服务（如阿里云OSS或AWS S3），并返回下载链接给用户。
